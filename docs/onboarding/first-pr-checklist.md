# First PR Checklist

- [ ] What changed is clearly summarized.
- [ ] Paths/modules touched are listed.
- [ ] Documentation was updated for behavior or ownership changes.
- [ ] `npm run lint` passed.
- [ ] `npm run test:run` passed.
- [ ] `npm run typecheck` passed.
- [ ] `npm run docs:check` passed.
- [ ] API/UI contract impact is explicitly stated (`none` or `explicit change`).
